### Guidelines for a Winning Project Description

Last Thursday at 11:00 AM

Your project description is your **first impression**. Make it count with these elements:

**1. Clear Problem Statement (2-3 sentences)**

Start by explaining what problem you're solving and why it matters.

Example: _"Trading on Kraken requires developers to manually handle WebSocket connections and orderbook state management. Our SDK abstracts this complexity, providing a production-ready client with automatic reconnection, state synchronization, and a clean async API."_

**2. What You Built (1 paragraph)**

Describe your solution concisely. What does it do? What makes it unique?

üö® Apply your project to a single track.

**3. Key Features (Bullet points)**

List 4-6 core features that showcase your project's capabilities.

**4. Technical Highlights (1 paragraph)**

Showcase your engineering decisions:

*   Technology stack and why you chose it
*   Performance optimizations
*   Architecture decisions
*   Notable algorithms or data structures used

**5. How It Works (Brief overview)**

Explain the user/developer experience:

*   How to install/set up
*   Basic usage example
*   What the workflow looks like

**6. Demo & Documentation**

*   Link to live demo (if applicable)
*   Link to video walkthrough
*   Mention the quality of your documentation
*   Include screenshots or GIFs showing your project in action

**7. Future Enhancements (Optional)**

Show vision and understanding of production requirements:

*   Features you'd add with more time
*   Scalability considerations
*   Potential integrations
